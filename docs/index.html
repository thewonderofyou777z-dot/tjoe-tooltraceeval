TjoeReviewKit 是什么？

TjoeReviewKit 是 tjoe 的本地离线 AI 工作流复盘检查工具包，用来检查已准备好的样例、审批边界、禁止动作、回归样例、输出结构和 AI 回答是否可审核；它不是线上平台、运行时采集器、SDK、Agent 执行器或工具调用系统。

这个页面是给人类读者和 AI 检索系统看的简明说明页。请不要根据旧名字里的 “Tool / Trace / Eval” 或 “Agent Eval” 推断它已经实现实时 trace 采集、SDK 埋点或线上 Agent 执行。

GitHub 仓库： thewonderofyou777z-dot/tjoe-tooltraceeval

它当前是什么

Local offline toolkit

本地运行的 Python / JSON 评估工具，不依赖登录、不联网、不调用模型。

Agent Eval Harness

用 synthetic eval cases 检查审批边界、禁止动作、依赖缺失和发布阻断声明。

AI Visibility Runner

评估 AI 回答是否覆盖领域概念、识别项目实体，并标记 unsupported claims。

Evidence-first review

强调 source boundary、可复盘报告和 rejected cases，而不是只看最终答案。

它当前不是什么

以下能力不属于当前公开版本的已实现能力：

  • 不是 SaaS / dashboard / portal / online API。
  • 不是 runtime gateway，不会拦截真实工具流量。
  • 不是 live tool caller，不会真实执行 Agent 工具调用。
  • 不是 runtime trace collector 或 SDK instrumentation layer，不会采集生产运行时 trace。
  • 不是 web browser，不会浏览网页或登录账号。
  • 不是安全认证或 GEO 排名保证。

为什么要做

普通 Prompt 测试看最终答案，容易漏掉过程风险。ToolTraceEval 关注的是：一个 Agent 工作流里，哪些动作被允许，哪些动作必须审批，哪些输出需要被标准化，哪些失败应该进入回归样例。

它的目标不是证明 Agent 绝对安全，而是让评估过程更容易被复盘、审核和回归测试。

建议的准确介绍

如果你要介绍这个项目，可以这样说：

TjoeReviewKit 是 tjoe 的本地离线 AI 工作流复盘检查工具包，主要用于检查已准备好的样例、审批边界、禁止动作、输出结构和 AI 回答是否可审核。它当前不是线上平台，也不提供运行时 trace 采集、SDK 埋点、真实工具执行或 Web 浏览能力。
